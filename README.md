# RimFix Constructor

Веб-конфигуратор автомобильных дисков. Позволяет выбрать тип кузова автомобиля, подобрать дизайн колёсных дисков и цвет их покраски, а затем автоматически наложить диски на изображение кузова с помощью OpenCV.js.

![RimFix Constructor](car_types/sedan.png)

## Возможности

- **8 типов кузовов** на выбор: седан, хэтчбек, купе, кабриолет, кроссовер, минивэн, пикап, универсал.
- **Библиотека дисков** — выбор дизайна колёсных дисков из встроенного набора.
- **Автоматическое определение положения колёс** с помощью OpenCV.js (template matching по шаблонному изображению).
- **Покраска дисков** — 18预设-цветов палитры (серебро, антрацит, чёрный глянец, бронза, золото, медь, розовое золото, титан и др.) плюс выбор собственного цвета через color picker.
- **Оригинальный цвет** — возможность оставить диск без покраски.
- **Адаптивный интерфейс** — отдельные раскладки для десктопа (боковая панель) и мобильных устройств (нижние sheet-панели с табами).
- **Тёмная тема** оформления.
- **Безсерверный запуск** — приложение работает прямо из файловой системы (двойной клик по `wheel_configurator.html`), никакой локальный HTTP-сервер и Python не нужны. Все изображения встроены в `assets.js` как base64 data URI, поэтому canvas не «загрязняется» и `getImageData()` работает корректно по протоколу `file://`.

## Используемые технологии

- **HTML / CSS / JavaScript** — интерфейс, логика и обработка изображений (всё в одном файле `wheel_configurator.html`).
- **OpenCV.js** — компьютерное зрение: поиск колёсных арок на изображении кузова через `cv.matchTemplate`. Библиотека полностью самодостаточна (WASM встроен в виде data-URI), внешних зависимостей нет.
- **assets.js** — все изображения проекта (кузова, диски, шаблон) в виде base64 data URI. Это позволяет приложению работать по протоколу `file://` без сервера: data URI считаются same-origin и не загрязняют canvas.
- **Python** *(опционально)* — простой локальный HTTP-сервер (`run_server.py`) для удобства разработки (CORS-заголовки, отключение кэша). Для запуска самого приложения сервер **не нужен**.

## Как это работает

1. Пользователь выбирает тип кузова — соответствующее изображение (data URI из `assets.js`) отображается на canvas.
2. Пользователь выбирает дизайн дисков из `wheel_types/`.
3. OpenCV.js выполняет template matching: изображение-шаблон (data URI из `assets.js`, исходник — `other_assets/replace_part.png`) ищется на изображении кузова, определяя позиции колёсных арок.
4. Выбранный диск масштабируется по размеру арки и накладывается на найденные позиции.
5. Если выбран цвет покраски — пиксели диска перекрашиваются с сохранением яркости (luminance), что сохраняет объём и рельеф диска.
6. Результат отображается на canvas в реальном времени.

## Установка и запуск

### Требования

- **Современный веб-браузер** с поддержкой Canvas и WebAssembly (для OpenCV.js).
- Python **не требуется** — сервер нужен только опционально для разработки.

### Быстрый запуск (без сервера)

1. Клонируйте репозиторий:

   ```bash
   git clone https://github.com/Siririn77/rimfix-constructor.git
   cd rimfix-constructor
   ```

2. Откройте файл `wheel_configurator.html` в браузере (двойной клик или через меню «Файл → Открыть»).

   ```bash
   # Linux/macOS — открыть в браузере по умолчанию:
   xdg-open wheel_configurator.html
   ```

3. Выберите кузов, диски и цвет — результат появится на canvas.

> Приложение полностью работает по протоколу `file://` (локально, без веб-сервера):
> WASM-бинарник OpenCV встроен прямо в `opencv.js`, а все изображения встроены
> в `assets.js` как base64 data URI (считаются same-origin), поэтому canvas не
> «загрязняется» и `getImageData()` работает корректно даже на файловой системе.

### Запуск через локальный сервер (опционально, для разработки)

Если при разработке удобнее работать через HTTP (CORS-заголовки, отключение кэша):

```bash
python3 run_server.py
```

Затем откройте `http://localhost:8000/wheel_configurator.html`.

> `run_server.py` раздаёт статику на порту `8000`, отправляет CORS-заголовки и
> отключает кэширование (удобно при разработке). Для обычного использования он **не нужен**.

## Использование

1. В боковой панели (на десктопе) или через нижние табы (на мобильном) выберите **тип кузова**.
2. Выберите **дизайн дисков** — диски автоматически найдут колёсные арки и наложатся на изображение.
3. Выберите **цвет покраски** диска из палитры или задайте свой цвет через color picker.
4. Для возврата к оригинальному цвету диска выберите «Оригинал» в палитре.

## Добавление новых типов кузовов и дисков

Типы кузовов и дисков задаются в JavaScript-массивах внутри `wheel_configurator.html`. Источником изображений служат data URI из объекта `ASSETS` (файл `assets.js`):

```javascript
const CAR_TYPES = [
    { id: 'sedan', name: 'Седан', src: ASSETS.car_types.sedan },
    // ...добавьте сюда новый объект
];

const WHEEL_TYPES = [
    { id: 'wheel1', name: 'Диск 1', src: ASSETS.wheel_types.wheel1 },
    // ...добавьте сюда новый объект
];
```

Чтобы добавить новый диск:

1. Поместите изображение PNG (с прозрачным фоном) в папку `wheel_types/` (например, `wheel3.png`).
2. Перегенерируйте `assets.js` (команда выше в разделе «Структура проекта»), добавив строку для нового колеса в генерирующий скрипт.
3. Добавьте запись в массив `WHEEL_TYPES`:

   ```javascript
   { id: 'wheel3', name: 'Диск 3', src: ASSETS.wheel_types.wheel3 }
   ```

4. Обновите страницу — новый диск появится в списке.

Аналогично для новых типов кузовов — изображение в `car_types/` + перегенерация `assets.js` + запись в `CAR_TYPES`.

> Рекомендуется использовать квадратные изображения дисков с прозрачным фоном и одинаковым разрешением.

## Разработка

Вся логика приложения — интерфейс, обработка изображений и покраска — находится в одном файле `wheel_configurator.html`; изображения встроены в `assets.js`. Для внесения изменений редактируйте `wheel_configurator.html`. После изменений достаточно обновить страницу в браузере (при открытии через `file://` — обычный Ctrl+R / F5; при разработке через `run_server.py` кэширование отключено автоматически).

### Ключевые части кода

- `ASSETS` (в `assets.js`) — все изображения в виде base64 data URI.
- `CAR_TYPES`, `WHEEL_TYPES`, `PALETTE` — данные кузовов, дисков и цветов.
- `loadOpenCV()` — динамическая загрузка OpenCV.js с автоматическим повтором при сбое.
- `applyWheelsWithOpenCV()` — поиск колёсных арок и наложение дисков.
- `getTintedWheelCanvas()` — покраска диска с кэшированием результата.
- `selectCar()`, `selectWheel()` — обработчики выбора.
